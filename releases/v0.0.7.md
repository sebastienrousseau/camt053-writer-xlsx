# camt053-writer-xlsx v0.0.7

First PyPI release of `camt053-writer-xlsx`, an Excel `.xlsx` writer
for camt053-parsed ISO 20022 bank statements. Aligned to the
camt053 suite v0.0.6 line.

## What's in the box

A single public function:

```python
from camt053_writer_xlsx import write_xlsx

write_xlsx(document, "out.xlsx")
```

where `document` is a `camt053.models.ParsedDocument` (the return
shape of `camt053.parse.statement_parser.parse_document`).

## Workbook layout

Four sheets, stable across releases:

| Sheet | Granularity |
| :--- | :--- |
| Metadata | One row per statement (header fields, account, balance / entry counts) |
| Balances | One row per balance |
| Entries | One row per `(entry, detail)` pair (flattened for pivot tables) |
| Reversals | Entries filtered to `reversal_indicator=True` or any return reason code |

Multi-detail entries are flattened: an entry with N transaction
details produces N rows that share the entry-level columns. The
first row of every sheet is bold so the workbook is readable in
Excel / Numbers / LibreOffice without further formatting.
